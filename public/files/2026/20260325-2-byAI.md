## 项目经验

### 模糊测试平台 - 架构迁移与 Web 化改造
**技术栈**：`React` / `Ant Design` / `ECharts` / `Webpack` / `Electron`
**角色**：前端负责人

主导模糊测试平台从 Electron 桌面端到 Web 平台的架构迁移项目，设计并实现多端代码复用方案：

1. **多端代码复用架构设计**：使用 Git Submodule 维护共享业务代码，通过环境变量区分运行环境编写兼容代码，利用 React Router 统一路由配置，通过 IPC 封装适配 Electron API 与 Node.js 模块，实现一套代码同时支撑 Web 与桌面端，减少约 40% 冗余代码，降低维护成本。

2. **高性能实时通信优化**：基于 WebSocket 实现控制指令双向通信，使用 SSE 流式传输测试进程日志。开源开发 [json-string-pack](https://www.npmjs.com/package/json-string-pack) NPM 包，对动态嵌套 JSON 进行压缩优化，将 9MB 数据压缩至 500KB；对固定结构消息采用 Protobuf 二进制序列化，减少 70% 以上的数据传输量，显著提升高并发场景下的响应速度。

3. **跨进程状态同步机制**：设计基于操作增量同步的状态同步协议，前后端双方预定义相同数据结构，通过 WebSocket 广播增量更新操作，实现前端与工具端状态的实时一致性同步，支撑测试过程中的实时校验功能。

4. **自定义网络报文处理**：基于 Node.js Buffer 字节流操作，扩展标准 pcap 协议格式，支持自定义报文头与数据段，满足业务特定数据存储需求，前端可直接解析和生成 pcap 文件。

5. **构建配置与开发体验优化**：简化 Webpack loader 配置，引入多线程并行压缩提升构建速度，配置前后端分离开发环境，提升本地开发调试效率；搭建本地模拟服务，脱离后端也可进行 UI 功能验证；通过 AI 上下文文档优化，减少 AI 工具重复扫描，提升 AI 辅助开发效率。

6. **动态测试报告生成**：基于 Paged.js 实现 HTML 转 PDF 报告功能，使用 HTML/CSS 控制报告样式，相比传统文档生成方案获得更高的可定制性，满足用户自定义报告需求。
